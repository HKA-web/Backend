1. Struktur Folder
	backend/
	├── bin/             # python portable (hasil ekstrak embeddable python)
	│   └── python
	│   		└──── python.exe
	├── manage.py
	├── settings.py
	├── urls.py
	├── asgi.py
	└── wsgi.py

2. Install Django

	klik install.bat

	setelah folder venv terbuat, klik cwd.bat, jalankan

	python -m pip install django


	Cek apakah berhasil

	python -m django --version


3. Buat Project Django Tanpa Subfolder

	Baru buat project

	python -m django startproject config .

	config hanya nama dummy untuk package awal, tapi karena pakai . maka Django tidak membuat folder baru, melainkan langsung generate file di backend/.

	Setelah itu folder akan jadi:

	backend/
	├── bin/
	│   └── python.exe
	├── manage.py
	├── config/
	│   ├── __init__.py
	│   ├── settings.py
	│   ├── urls.py
	│   ├── asgi.py
	│   └── wsgi.py

4. Pindahkan File ke Root (Opsional)

	move config\settings.py .
	move config\urls.py .
	move config\asgi.py .
	move config\wsgi.py .

	Hapus folder config/ setelah isinya dipindahkan.

	Edit manage.py dan update baris:

	os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'settings')

	(sebelumnya config.settings → jadi settings).

	Edit asgi.py dan wsgi.py juga, ganti:

	os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'settings')

5. Jalankan Server
	
	python manage.py runserver

📌 Dengan cara ini, kamu punya Django project minimalis dengan file inti (settings.py, urls.py, asgi.py, wsgi.py) langsung di root backend/, dan Python portable ada di bin/.

============================ 

Kalau nanti kamu mau update Python ke versi terbaru tapi tetap pakai struktur portable seperti sekarang (bin/python.exe + Django project di root), alurnya bisa kamu anggap kayak ganti mesin baru untuk project lama.

🔄 Langkah-langkah Update Python Portable (versi baru)

1. Download Python Embeddable terbaru

	Ambil dari python.org/downloads/windows

2. Backup / Replace bin/

	Simpan dulu folder backend/bin/ lama (buat jaga-jaga).

	⚠️ Kalau kamu rename folder baru jadi bin/ → pastikan manage.py dan script BAT kamu tidak perlu diubah.

3. Reinstall pip

	Download get-pip.py lagi (atau pakai yang lama).

	Jalankan:

	python get-pip.py

	Cek:

	python -m pip --version

4. Reinstall dependencies

	Karena bin/ baru kosong, semua package (termasuk Django) hilang.

	Cek project lama, kalau kamu sudah pernah freeze dependencies:

	python -m pip install -r requirements.txt

	Kalau belum, install ulang Django manual:

	python -m pip install django

5. Jalankan migrasi (opsional)

	Database db.sqlite3 aman, tidak hilang.
	Tapi sebaiknya jalanin:

	python manage.py migrate

6. Tes runserver
	python manage.py runserver

	Kalau semua lancar → project jalan di Python baru 🎉

📦 Best Practice Tambahan

Selalu simpan requirements.txt

python -m pip freeze > requirements.txt

Jadi update Python gampang → tinggal install ulang semua paket sesuai versi.

Pisahkan bin/ dari kode project
Misalnya:

backend/
├── bin/        ← Python portable
├── manage.py
├── settings.py
├── urls.py
├── asgi.py
├── wsgi.py
└── requirements.txt


Jadi kalau update Python, cukup replace bin/.

Cek kompatibilitas Django
Misalnya Django 5.1 butuh Python >= 3.10. Jadi pastikan versi cocok.
